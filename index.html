用Github+Hexo搭建个人站教程

发表于 2017-01-20 |

一直拖着没写教程，室友也想建一个，那这次就把这个任务给补了吧。
先送上一首我最爱的黄老板的歌，活跃一下气氛。

既然是教程，那就得接地气。所以我希望自己写的东西，只要是识字的人都能看得懂。

那就开始动手吧!

第一步，建立Github博客域名

（建立域名就像给房子选个地址，弄个门牌号，告诉大家这里能找到你。）

  1. 打开->GitHub

  2. 注册

    • 依次输入username（用户名）、Email Address（邮箱）、Password（密码）
    • 然后点击Create an account（创建账户）
    • 直接按Continue（我因为免费所以用GitHub的域名的）
    • 最后按skip this step（跳过吧，当然你也可以认真填一下）
    • 然后去你的邮箱验证一下账号。

  3. 回到GitHub，在最上面的导航栏找到“+”这个符号，然后点击New repository（新建一个仓库）

  4. 写入仓库名（Repository name）

    • Repository name需要是这样的：

      你的用户名.github.io（例如我的用户名是lisonzhao，那么我的就是：lisonzhao.github.io，搞错就没有用的嗯）

  1. 然后create reposity(其它的先不管，有必要的时候我相信你会去了解的)

到此为止，你的域名就选好了！

第二步，搭好博客框架Hexo

（现在开始打地基造房子）

  1. 打开-> hexo官方中文文档
    官方文档写的很好，但是需要有耐心，按照他的步骤慢慢来。

  2. 先下载好git和Node.js

  3. 在任意位置单击右键，选择“Git Bash Here”（默认Windows情况）

  4. 输入命令按回车键，安装hexo
    $ npm install -g hexo-cli

  5. 然后选择或创建一个文件夹（放个人站的内容，这里就是你的写作基地），然后在这个文件夹中任意位置右键打开“Git Bash Here”，输入命令后按回车键，初始化hexo。
    hexo init

  6. 完成后，文件夹是这样一个情景

第三步，配置hexo

  1. 打开_config.yml文件(建议使用下个sublime耍耍，用记事本打开也是没关系的)

  2. 找到url和deploy标签：，填上这些内容(其它配置参照hexo官方文档)

  3. 保存后，右键打开Git bash,输入命令并回车
    npm install hexo-deployer-git --save

    到这里配置已经完成

  4. 在文件夹中右键打开Git bash，输入命令并回车
    （该指令以后会经常用到，作用是生成页面，可以简写为 hexo g）
    hexo generate

  5. 输入指令
    (同样很重要的指令，作用是将本地博客提交到Github，可能需要输入Github的密码，可以简写为 hexo d)
    hexo deploy

  6. 在浏览器中输入http://你的用户名.github.io/（例如我的：lisonzhao.github.io），就能看到你的个人站诞生了！

第四步，使用NexT主题美化

（造好了房子，现在开始装潢）

  • hexo可以用很多主题希望大家能选择一个自己喜欢的。我比较喜欢简单，所以选择NexT主题。

  • 使用->NexT官方教程根据你自己的偏好进行配置，请仔细阅读。
    尤其注意 _config.yml 配置文件,hexo和NexT中各有一个，要分清楚配置哪一个！

第五步，开始写你的第1篇博文

（添置家具咯！）

  1. 新建一个文章
    $ hexo new "第1篇博文"

  2. 在source/_posts文件夹中找到“第1篇博文.md”
    用记事本打开进行书写，推荐一个在线Markdown写作编辑平台，写完后进行保存。

  3. 回到个人站文件夹,右键打开Git Bash, 执行三个命令（以后每一次更新或者提交文章都需要执行这三个指令。）
    $ hexo clean （这个可以不用，遇到一些故障时，可以先用这个试一下。）
    $ hexo g
    $ hexo d

4.浏览器中输入http://你的用户名.github.io/，新的文章就会出现了

写教程比自己折腾要累很多，差不多就这样了，若有其他问题，请直接联系我哦！
